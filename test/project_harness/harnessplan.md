# Rust Port Test Harness Plan

## Purpose
Integration tests that exercise every ported Ratpack function with shared test
cases (JSON format). Compares Rust results against known-good reference values
generated by the C++ harness or derived from mathematical identities.

## Architecture
```
test/project_harness/
├── harnessplan.md           ← this file
tests/
├── ratpack_integration.rs   ← main integration test file
└── (additional test modules as needed)
test/test_cases/
├── testplan.md              ← test case inventory
├── arithmetic.json          ← arithmetic test cases
├── conversion.json          ← string ↔ number roundtrips
├── exp_log.json             ← exp, log, pow, root
├── trig.json                ← sin, cos, tan + hyperbolic
├── itrig.json               ← inverse trig + inverse hyperbolic
├── factorial.json           ← factorial / gamma
├── logic.json               ← bitwise AND/OR/XOR, shifts
├── support.json             ← int_rat, frac_rat, gcd_rat, etc.
└── reference_outputs/       ← C++ harness-generated reference data
```

## Test Strategy

### Level 1: Mathematical Identity Tests (no C++ reference needed)
These tests validate the Rust implementation using known mathematical truths:
- `exp(ln(x)) ≈ x` for x > 0
- `sin²(x) + cos²(x) ≈ 1`
- `a + b = b + a` (commutativity)
- `(a * b) / b ≈ a` (inverse)
- `int(x) + frac(x) ≈ x`
- `string_to_number(number_to_string(x)) ≈ x` (roundtrip)

### Level 2: Known Value Tests (mathematical constants)
Compare against well-known values (not requiring C++ build):
- `sin(π/6) ≈ 0.5`
- `cos(0) = 1`
- `ln(e) = 1`
- `10! = 3628800`
- `√4 = 2`
- etc.

### Level 3: C++ Reference Tests (require port_harness build)
Run the same test inputs through both C++ and Rust, compare outputs.
These are the most rigorous — they catch subtle precision/rounding differences.

## Test Case JSON Format
```json
{
  "id": "add_001",
  "function": "add_rat",
  "description": "Basic integer addition",
  "inputs": {
    "a": "3",
    "b": "4"
  },
  "params": {
    "radix": 10,
    "precision": 128
  },
  "expected": {
    "result": "7",
    "error": null
  },
  "tolerance": null,
  "tags": ["arithmetic", "basic", "level2"]
}
```

## Running Tests
```powershell
# Run all integration tests
cargo test --test ratpack_integration

# Run only arithmetic tests
cargo test --test ratpack_integration arithmetic

# Run only tests that don't need C++ reference
cargo test --test ratpack_integration -- --skip cpp_reference
```

## Acceptance Criteria
1. All Level 1 (identity) tests pass
2. All Level 2 (known value) tests pass within stated tolerance
3. All Level 3 (C++ reference) tests match to within 10^(-precision/2) relative error
4. No panics or undefined behavior
5. All error cases produce the correct CalcError variant
